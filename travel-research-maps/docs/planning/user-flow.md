# travel-research-maps 用户流程

## 文档目的

描述用户和 Codex 操作者如何完成一次 Firecrawl 多平台旅行地点研究、审核清单确认和可选 Google Maps 写入。

## 适用范围

适用于用户触发 `$travel-research-maps` 后的交互流程。本文档仅根据当前仓库可见内容整理；未在仓库中找到证据的内容均不做假设。

## Plan 或项目证据

| 证据 | 流程含义 |
| --- | --- |
| `SKILL.md` 必需输入 | 流程入口需要目的地和旅行日期或时间范围。 |
| `SKILL.md` 工作流 2 | Firecrawl 是主研究路径，并按目的地区分平台优先级。 |
| `SKILL.md` 工作流 6 | 审核清单是默认交付物。 |
| `SKILL.md` 工作流 7-9 | Google Maps 保存需要用户批准、唯一匹配和保存结果核对。 |
| `scoring-rubric.md` 审核输出 | 审核表需要分开优先去、备选、排除/待确认。 |

## 入口和出口

| 类型 | 描述 |
| --- | --- |
| 入口 | 用户询问旅游目的地、旅行计划、旅游景点、美食推荐或要求整理到 Google Maps。 |
| 默认出口 | 输出中文审核清单，等待或不等待用户后续批准。 |
| 批准后出口 | 报告 Google Maps 已保存、跳过和待确认项目，并保留列表页面作为交付页。 |
| 阻塞出口 | 报告 Firecrawl 不可用、平台限制、覆盖不足、匹配失败或 Google Maps 不可用等原因。 |

## 主流程

1. 识别请求是否匹配旅行研究和地图收藏。
2. 从请求中提取目的地和旅行日期或时间范围；缺失日期时标记为“计划前往日期未知”。
3. 读取 `references/scoring-rubric.md`。
4. 根据目的地选择平台优先级。
5. 使用 Firecrawl 搜索公开内容，处理结果后提交 `firecrawl_search_feedback`。
6. 对候选 URL 使用 Firecrawl scrape 或 extract 获取正文和结构化证据；必要时使用 map 或最后 fallback agent。
7. 分别检查景点和餐厅覆盖门槛。
8. 将候选证据整理为 JSON，运行评分脚本并要求 2 个正向平台门槛。
9. 对优先去和备选地点进行事实核验，并用 Google Maps 核验地址、营业状态和风险。
10. 输出中文审核清单。
11. 请求用户明确批准地图写入和目标列表。
12. 获得批准后，使用 Google Maps 搜索并保存唯一匹配地点。
13. 核对列表数量，报告保存结果。

## 备用流程和失败流程

| 条件 | 行为 |
| --- | --- |
| 用户未给日期 | 继续研究，但标记“计划前往日期未知”，季节性、临时闭馆和预约信息出行前复核。 |
| Firecrawl MCP 不可用 | 报告阻塞原因，不自动退回内置 web search。 |
| 平台页面无法公开抓取 | 不计入浏览量或评分证据，记录覆盖缺口。 |
| 无法识别作者、日期或推荐立场 | 不计分，只保留为缺口或待确认线索。 |
| 发现广告或商业合作 | 排除，不计分也不计来源数。 |
| 景点或餐厅覆盖门槛未达成 | 停止评分、事实核验、审核清单和地图写入，只报告缺口。 |
| 用户未批准 Google Maps 写入 | 结束于审核清单。 |
| Google Maps 匹配不唯一 | 不保存该地点，报告原因。 |

## 权限或角色差异

| 角色 | 权限 |
| --- | --- |
| 用户 | 提供目的地、日期、可选列表名；批准或拒绝地图写入；处理 Google Maps 登录、验证码和授权。 |
| Codex agent | 使用 Firecrawl 收集公开内容、整理证据、运行评分脚本、输出审核清单。 |
| Codex agent 批准后 | 在 Google Maps 中保存唯一匹配地点。 |

## UI/CLI 状态和文案敏感决策

- 审核清单必须清楚标注“优先去”“备选”“排除/待确认”。
- 写入地图前必须明确请求人工批准。
- 覆盖不足时不能输出部分地点清单，以免用户误以为研究完整。
- 对未确认信息使用“未确认”或“待确认”，不能写成事实。

## 验收场景

| 场景 | 验收 |
| --- | --- |
| 正常研究并输出清单 | 清单分为景点和餐厅，包含评分、来源数、正向平台、可信度和待确认事项。 |
| Firecrawl 不可用 | 报告阻塞，不输出伪完整清单。 |
| 覆盖不足 | 报告已浏览内容数、候选数和具体缺口，不输出部分清单。 |
| 用户批准写入 | 仅保存名称和地址或城市唯一匹配的地点。 |
| 用户拒绝或未回应批准 | 不写入 Google Maps。 |

## 非目标

- 不提供路线排序或每天行程安排。
- 不自动关注、点赞、收藏、评论、私信或发布内容。
- 不绕过平台安全机制。
