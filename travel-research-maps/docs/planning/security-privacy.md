# travel-research-maps 安全与隐私说明

## 文档目的

记录 Firecrawl 多平台研究、浏览器会话、外部写入、证据收集和数据最小化边界。

## 适用范围

适用于 Firecrawl 公开内容研究、Google Maps 核验与保存、评分脚本和项目文档维护。本文档仅根据当前仓库可见内容整理；未在仓库中找到证据的内容均不做假设。

## Plan 或项目证据

| 证据 | 安全或隐私要求 |
| --- | --- |
| `SKILL.md` 工作流 2 | Firecrawl 是主研究路径，不绕过登录墙、验证码、付费墙或反爬机制。 |
| `SKILL.md` 工作流 2 | Firecrawl 不可用时报告阻塞，不自动退回内置 web search。 |
| `SKILL.md` 工作流 7-9 | Google Maps 写入必须等待用户明确批准。 |
| `README.md` Notes | 仓库不包含 API key、token、Cookie、账号密码或其他凭据。 |
| `score_candidates.py` | 评分器不执行网络请求。 |

## 敏感数据和数据分级

| 数据 | 分级 | 处理要求 |
| --- | --- | --- |
| 用户账号 Cookie、密码、token、本地存储 | 高敏感 | 不读取、不记录、不写入文档、代码、日志或响应。 |
| Firecrawl 搜索和抓取结果 | 研究证据 | 仅用于旅行证据审计和评分；不可公开访问内容不计分。 |
| 作者或频道、URL、发布日期、推荐立场 | 研究证据 | 仅用于旅行证据审计和评分。 |
| Google Maps 列表名和地点保存结果 | 用户操作数据 | 仅在用户批准后使用，并报告保存、跳过、待确认状态。 |
| 目的地和旅行日期 | 用户请求数据 | 用于研究范围和事实核验；日期未知时显式标记。 |

## 认证和授权

- Firecrawl MCP 的可用性由当前 Codex 工具环境提供；skill 不要求用户输入 Firecrawl secret。
- 用户负责完成 Google Maps 的登录、验证码和授权。
- Codex agent 只能在用户批准后使用已登录 Google Maps 浏览器会话保存地点。
- 遇到任何平台登录页、验证码、付费墙或安全校验时，必须停止相关路径并记录覆盖缺口。

## Secrets 处理

- 不向用户索要或记录账号密码、Cookie、token、私钥或 `.env` 值。
- 不把任何凭据写入 `evidence.json`、`scored.json`、规划文档、代码或测试。
- 若未来需要配置外部服务，必须通过环境变量或用户明确批准的本地配置路径处理，并避免输出具体 secret 值。

## 信任边界

| 边界 | 规则 |
| --- | --- |
| Codex 到 Firecrawl | 只请求公开可访问内容；搜索后提交简洁 feedback，不包含大段页面内容。 |
| Firecrawl 到平台页面 | 不绕过平台访问控制，不提交互动表单。 |
| Codex 到 Google Maps | 评论只用于风险、营业状态和地点匹配核验；写入列表前必须批准。 |
| Workflow 到评分脚本 | 脚本只接收已整理的本地 JSON，不直接访问网络。 |

## 审计和日志预期

- 审计数据应记录景点和餐厅分别已浏览内容数、每篇 URL、来源平台、作者或频道、日期和候选数。
- 审核输出只展示最能代表正向或风险信号的 2-3 条链接，保留全部证据审计数据。
- 不记录浏览器会话凭据或无关页面内容。

## 隐私约束

- 不读取与旅行研究无关的浏览器标签页内容。
- 不读取浏览历史、密码管理器、Cookie、本地存储或账号设置。
- 不主动上传私人数据到替代账号或第三方服务。

## 威胁、缓解措施和剩余风险

| 风险 | 缓解措施 | 剩余风险 |
| --- | --- | --- |
| 误把营销内容当作独立推荐 | 按广告、赞助、商业合作、探店邀约、店方账号等规则排除。 | 未披露商业关系的内容仍可能存在偏差。 |
| Firecrawl 抓取覆盖不足 | 记录平台、候选和内容数量缺口；未达门槛不输出清单。 | 平台限制随时间变化，真实覆盖需逐次验证。 |
| 未经批准写入 Google Maps | workflow 明确要求审核清单后请求人工批准。 | 实际浏览器自动化执行仍需操作者遵守。 |
| 地点实时状态变化 | 事实核验优先官方来源，并标记待确认项。 | 营业、预约和闭馆信息仍可能在出行前变化。 |

## 实现指引

- 新增外部服务或网络调用前，必须更新本文件和 `SKILL.md`。
- 自动化失败时优先停止并报告，不应增加绕过逻辑。
- Firecrawl feedback 中不得放入 secrets 或大段原始页面内容。

## 验收标准

- `SKILL.md` 明确禁止绕过平台限制。
- Google Maps 写入路径继续要求用户批准。
- 文档和测试中不包含真实账号、Cookie、token、私钥或 `.env` 值。
