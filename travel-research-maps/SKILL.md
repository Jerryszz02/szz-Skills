# 旅行研究与地图收藏

为给定目的地制作可审核的旅行地点清单。默认输出中文，不使用固定旅行偏好。

## 必需输入

先确认或从请求中提取：目的地、旅行日期或时间范围。可选输入为语言、指定来源和已有 Google Maps 列表名。

若用户未提供日期，使用“计划前往日期未知”，仍研究研究日向前 365 天内的内容；将季节性、临时闭馆和预约信息标为需要在出行前复核。

## 工作流

1. 阅读 `references/scoring-rubric.md`，按其规则收集和标注证据。
2. 使用 `chrome:control-chrome` 从小红书收集研究日向前 365 天内的旅行内容，再进入后续评分与事实核验流程。先列出用户 Chrome 标签页，按可见标题、URL 与标签页活跃状态精确认领已登录的小红书标签页；对每个目的地先站内搜索“目的地 + 旅游攻略”，据此确定景点和餐厅候选，再对每个候选搜索“地点名称 + 目的地”。不得使用公开检索、Computer Use 或应用内浏览器替代 Chrome 访问小红书。仅在小红书覆盖不足时，补充其他独立旅行或社交内容。不要把 Google Maps 评论计入推荐分；只用它核验风险信号、营业状态和地点匹配。
   - Chrome 控制仅可输入站内搜索词、打开公开帖子、滚动并读取可见的结果和正文；不得读取 Cookie、密码、本地存储、历史记录或其他与研究无关的数据；不得点赞、收藏、关注、评论、私信、发布内容或提交任何表单。
   - 若无法认领已登录的小红书标签页，或出现登录页、验证码或其他安全校验，立即停止小红书路径，记录覆盖缺口并由用户完成账号、密码、验证码和任何授权步骤。不得尝试绕过或改用其他浏览器自动化路径。
   - 只有能打开并识别作者、日期和立场的帖子才可作为评分证据；搜索结果摘要、转载聚合页和无法打开的链接不得计分。排除广告、返佣、商业合作、探店邀约及店方账号。
   - 若小红书或其他平台受登录墙、验证页、付费墙或反爬机制限制，记录覆盖缺口；不得尝试绕过。
   - 将“浏览一篇帖子”限定为：打开帖子正文，并识别作者、发布日期和推荐立场；仅看到搜索结果摘要、转载聚合页或无法打开的链接均不计入浏览量。
   - 景点与餐厅分别满足硬性覆盖门槛后，才可进入评分：景点至少浏览 10 篇相关帖子并形成至少 10 个不同景点的候选清单；餐厅至少浏览 10 篇相关帖子并形成至少 5 家不同餐厅的候选清单。两类帖子和候选数量不得相互抵扣。
   - 在审计数据中分别记录景点/餐厅的已浏览帖子数、每篇 URL、作者、日期及最终候选数。任一门槛未满足时，停止后续评分、事实核验、审核清单和地图写入；只报告覆盖不足与具体缺口，不得输出部分地点清单。
3. 为每个候选记录名称、类别、链接、作者、日期、推荐档位、是否营销、内容指纹和一句摘要。排除广告、赞助、商业合作、探店邀约、店方/品牌账号、优惠码或导购内容。
4. 确认上述景点与餐厅覆盖门槛均已满足后，将候选证据写成 JSON，运行：

   ```powershell
   python scripts/score_candidates.py --input evidence.json --output scored.json --as-of YYYY-MM-DD
   ```

   在当前日期不等于出行日期时，`--as-of` 使用研究当天日期。脚本只做去重、计分和分层；浏览与事实核验由本工作流完成。
5. 对“优先去”和“备选”地点核验事实。门票、预约、营业/闭馆信息优先使用官方来源；官网缺失时才使用预约平台、场馆官方社媒、Google 商家页或近期旅行平台，并标记为“非官方核验”。
6. 先输出审核清单，禁止在此阶段写入 Google Maps。只输出“景点”和“餐厅”两组地点，不提供按天、按时段或路线式行程。每项使用以下固定格式：

   ```markdown
   **地点名称**｜开放/营业：时间或“未确认”｜预约：需要/建议/不需要/未确认
   一句话介绍。
   附加：景点写门票、最后入场、闭馆日、官网与小红书证据；餐厅写菜系、招牌、预算、地址/街区、官网或订位渠道与小红书证据。标注净分、独立正向来源数、信息可信度和待确认事项。
   ```

   按评分层级排序，但不要另行生成“第几天去哪里”的建议。对无有效社媒证据的地点标为“待确认”，不得因评分不足暗示其不值得前往。
7. 明确请求人工批准。默认全选“优先去”，用户可回复“加入全部，排除 A、B；使用列表 X”或要求新建“目的地·日期”列表。未得到批准时，结束于地点审核清单。
8. 获得批准后，使用已登录的 Google Maps 浏览器会话。对每项搜索“名称 + 目的地”，核验名称和地址/城市一致后保存到用户指定或新建的列表。结果不唯一、关闭或无法匹配时不保存，并报告原因。
9. 核对列表内已保存的数量，报告已保存、跳过、待确认项目。保留该列表页面作为交付页。

## 审核与安全边界

- 不得把未核验的搜索摘要当作帖子证据；无法打开或识别内容时降低覆盖度并说明缺口。
- 同一作者、转载或相同素材只算一个独立来源。优先项目不代表保证可订位或实时营业。
- 出现永久关闭、食品安全或多来源一致的重大服务问题时，标为排除，不进入默认地图写入范围。
- Google Maps 写入是外部副作用：仅在用户明确批准审核清单及目标列表后执行。不要读取 Cookie、密码或其他浏览器会话数据。
- 无法使用 Google Maps 或用户未登录时，保留审核清单并说明阻塞原因；不要尝试替代账号或上传私人数据。
